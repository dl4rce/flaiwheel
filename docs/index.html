How Search Works

Hybrid retrieval with optional cross-encoder reranking for maximum precision.

1

Vector Search (ChromaDB + HNSW)

Your query is embedded and matched against document chunks using cosine similarity. Finds semantically related content even when exact words differ.

Semantic
2

BM25 Keyword Search

In parallel, exact keyword matching with TF-IDF weighting catches results that share specific terms with the query — even if semantically distant.

Keyword
3

Reciprocal Rank Fusion (RRF)

Both result lists are merged using RRF with configurable weights and k parameter. Documents found by both methods get boosted.

Configurable
4

Cross-Encoder Reranking

A cross-encoder model reads each (query, document) pair together and produces a precise relevance score. Solves vocabulary mismatch — "auth bypass" finds "client-side auth flag".

Optional · +50ms
5

Relevance Filter & Return

Results below the configurable minimum relevance threshold are dropped. Only the most relevant chunks reach the AI agent.

Zero noise

How It Works

One command to install. Your AI agent handles the rest.

1

Install

Run the install command from your project directory. It creates a private knowledge repo, starts a Docker container, and connects your AI agent via MCP.

2

The AI searches before coding

Before every code change, the agent searches Flaiwheel for architecture decisions, past bugfixes, API docs, and best practices. Context the source code alone cannot provide.

3

The AI documents after fixing

After every bugfix, the agent writes a structured summary (root cause, solution, lesson learned). It is instantly indexed and searchable.

4

Knowledge compounds

The next search finds the previous fix. The next bug is cheaper. The flywheel accelerates. Every task makes the system smarter.
